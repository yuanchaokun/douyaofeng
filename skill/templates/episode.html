Chapter 01

底线被抬高了

水位涨上来了 — 原来的山头变成浅滩，新的高峰还在远方

本期开场摇摇就抛了一个最近卡在脑子里的观察 — AI 把"产出质量的默认水平"抬高了。

他举了一个生图的例子：之前没有 SD 模型那会儿，要生图必须把画面、构图、镜头、光线写得很细很细；现在你简简单单一句话，出来的结果也"不至于不能接受"。如果想要更好的，再加复杂提示工程。

「就我也不知道怎么形容了 — 大概是说原先的时候你不好好干活，你就饿死了；现在你不好好干活，至少能不饿死。」

— 摇摇
白
社会平均水平。

这件事乍听不重要，其实是本期所有讨论的暗线 — 当所有人的"默认产出"都被工具抬到一个新的中位数，你之前以为可以靠的护城河（埋头钻一点，比别人深一寸）就不太够了。

Chapter 02

简历名词堆砌的反面

摇摇这周继续在面试多 agent 架构的人。他发现一个很普遍的模式 — 很多简历就是名词的堆砌。

「他在里头写了一堆 — 意图识别、任务派发、路由、RAG 知识库、LangChain、LangGraph。但你深入去问的话，反而说不清楚 — 模棱两可，像是直接用豆包翻译了一下这个词。从他这里接收不到新东西。」

— 摇摇

他给出了一个反向的、自己以后写简历也想用的方法：

问越多名词，越容易踩到对方的非强项

"你把这些词问的多了，势必会命中到他的非强项或不擅长的领域，效果反而不好。"

用一个项目讲清楚整条链路

"任务从哪里开始 → 怎么实现 → 最终产出一个什么结果 → 有什么效果上的提升 — 我觉得这样比堆十个名词管用。"

这其实和本期暗线呼应 — 当 AI 把每个名词的"表面熟悉度"都拉高之后，"我把一件事完整做出来"成了越来越稀缺的证明。

Chapter 03

几百个技能包 × Skill 路由

摇摇现在的主活是做意图识别和 skill 管理。他们大部门已经沉淀出来好几百个技能包（skills），完全没法一股脑挂上去 — 上下文会爆，描述太相近的 skill 也会分派错乱。

他目前的解法是：

Skill 描述做成知识库

把每个 skill 的 description + 全文当作可检索的知识库。

意图识别后再检索

任务进来 → 先意图识别 → 用关键词去 skill 知识库里检索 → 调用对应 skill。

承认这是基建不够好的妥协

"还是因为模型能力还不够强 — 等模型再强一点，可能直接挂载就好了。"

「最重要的还是 context 和 token 管理目前的基建还不够好，所以才需要这样的架构来做。」

— 摇摇

Chapter 04

赚钱的本质是信息差

白豆引用了最近读的一篇文章 — 关于"赚钱 = 信息差"的四象限：

「无论是做东西也好、做咨询也好，都需要有一个基础的信息差 — 你知道别人不知道。如果这个市场所有人都很聪明、都知道，你就卖不出去。但当你跳出这个市场，把这件事传达给那些不知道的人，那将会是非常赚钱的，而且是真的能产生价值的。」

— 白豆（转述某博主的观点）

摇摇当场把它翻译成了一个更具体的版本：

摇
是不是有点像寻找细分市场？你自己在英语国家教英语不好赚钱，但是在非英语国家、并且出国氛围比较热烈的那种国家去教英语 — 就很赚钱。
白
可以这样理解。

核心动作只有两步 — 比别人快一步，然后找到具体快于谁的那群人。

Chapter 06

比上班还累，但累得不一样

李墨玩开场说了一句很有意思的话 — 他这周回顾了一下，发现：

「我现在工作强度比我上班的时候还累。但累的原因不一样 — 不是『为了只赚一份钱去干活』，是自己真的感兴趣、想干。是自愿的那种类。跟上班那种还不太一样，但身体上表现出来都是疲惫 — 原因不同。」

— 李墨玩

这件事和上一期他说的"兴趣本身就是结果"是同一条线 — 当一件事的"为什么"变了，疲惫的颜色也会变。但身体不会撒谎，疲惫还是会显现出来。

他这周想做的一件正面挑战是：5 月份要拍一个露脸的视频。

「我上个月不是立了个 flag 嘛 — 这个月要拍一个露脸的视频。对我来说挑战挺大的 — 每个人不一样，有的人让他直个播他表现很自然；我就不太行，不太习惯露脸，这是性格决定的。」

— 李墨玩

Chapter 07

唱跳 Rap 都得会一点

一个发光的圆心 — 不在某一条线上做到 0.1%，而是横向都会一点

这是本期最被点名的金句 — 李墨玩讲他这段时间做自媒体的最大体会：

「你去做个人的自媒体博主，有点像你是什么唱跳 Rap，什么都得会一点 — 你懂我意思不？但你不一定要把它做的非常精。至少懂得怎么去做视频、怎么写、怎么演讲。」

— 李墨玩

他把这件事拍到了职场里：

「我觉得上班来说，职场必备的两个技能 — 一个是写作，一个是演讲能力。我虽然没有上班，但这两个我已经都在练。以前没那么觉得，现在越来越觉得这个东西很重要 — 而且还挺不好干，你没有一定的实践积累，是干不好的。」

— 李墨玩

之前已经有人给他反馈 — "我去看你去年发的文章，你进步挺多的"。这种来自陌生人的、隔了一年的回头看，是他最有动力的那种回响。

Chapter 08

四个平台，四种逻辑

李墨玩这段把他这段时间观察各个平台的体感梳理了一遍 — 同一份内容在不同平台命运截然不同：

小红书
推流权重看内容、不看粉丝；强调真实感、素人；不希望任何一个博主做成山头。商业化克制。
B 站
鼓励"做大的 UP 主"；扶持新人；视频要"精雕细琢"；数据看板做得好。封面要用心，同样的封面在小红书是爆款逻辑，在 B 站可能只有 9 个阅读。
抖音
分发优先粉丝、再到推荐；体量大；鼓励博主积累粉丝、做电商和直播；扶持计划多。
公众号
粉丝权重最高 — 一个公众号粉丝 ≈ 十几个抖音粉丝。连接墙最紧密，跟微信直接打通。

他给了一个反直觉的判断 — "几十万粉丝的博主一年到头赚不了多少钱，真正能转化的是那些只有几万粉丝、但黏性极强的账号。" 粉丝数有时候是虚荣指标，转化能力才是真硬通货。

「DontBeSilent 那种，几万粉丝，一年收入比很多百万粉丝博主都高 — 而且看起来还更轻松。但他的活也不好干 — 运营社群、给别人解答问题，是脑力劳动。」

— 李墨玩

Chapter 09

走路时截图：碎片化的选题库

不是某一次大块的努力 — 是每天散落的零碎积累

李墨玩拆解了他"零碎时间堆素材"的工作流 — 不要一次性憋一整篇文章，而是平时分散积累：

定个选题 — 不急写

比如最近想写一篇"小程序怎么上线"，先把这个选题挂在选题库里。

每做一步就截图扔进对应文档

"我今天每做一步申请，我就截图放过来。最后我再去写这篇文章的时候，素材就积累的差不多了。"

走路时的小红书截图也是素材

"我刚刚的截图都是走路时刷小红书顺手截的 — 不是用额外的大块时间去专门做这件事情。"

陌生人打赏也是选题

"我顺手把这个东西记录下来 — 万一后面成了个什么大 V，我回顾的时候可以用上这些素材。梦想还是要有的。"

他还有一个更细的产品取舍 — 十几个主题里只把两个做成付费，剩下都是免费。"做这种东西要有取舍，不能为爱发电，但也不要把所有都收费 — 大部分人来说免费的够用。"

Chapter 10

白豆的 DHH 时间

25 岁才学编程的赛车手 — 99% → 99.5% 太贵，跨界更划算

白豆这周最沉迷的是 DHH 的 Lex Fridman 访谈（六小时）。DHH 是 Ruby on Rails 的作者、37signals 的合伙人、勒芒 24 小时拉力赛的冠军、好几本书的作者、丹麦人。

25 岁才正式开始学编程

"之前会有一些接触，但觉得非常受挫 — 因为他觉得自己并不像那些天才一样。直到邂逅了 Ruby — 觉得这东西很优雅，像写诗一样，比其他编程语言更让他上头。"

跨界比再深一寸更划算

"你不一定要在一个领域做到前 1% 或者 0.1%。99% → 99.5% 那一点点需要付出几倍的努力 — 但你可以跨界做很多其他的事情。"

跨界做到极致：赛车 + 商业 + 工程 + 写作 + 家庭

他既是赛车冠军、又是合伙人、又是顶级工程师，写过 Rework、Remote 等几本书 — "家庭也非常和睦，又是一个非常好的父亲。"

4.6 模型出来之前：坚持古法手工编程；出来之后：积极拥抱 agent

"他对每件事都有他自己的判断 — 不是为了反对工具而反对，是为了真的觉得 Ruby 写得像诗才用 Ruby。"

白豆说看完这两个视频，"获取一种很好的感染力"。这种感染力其实和摇摇说的"底线被抬高"是同一件事 — 当工具把"勉强能做"的边界扩到所有人都覆盖时，真正稀缺的反而是「同时能在好几件事上做得很好」的人。

Chapter 11

37signals 的产品哲学

买断终身、开源、强制休假 — 一种刻意被克制的商业模式

白豆顺势体验了 37signals 旗下的几个产品 — 邮箱、日历、看板。他被打动的不是某个具体功能，而是这套产品哲学的一致性：

邮箱 HEY：第一封信就是 "Block Sender"

新邮件来时不直接进收件箱 — 你先决定是"拉黑这个人"还是"继续接收"。验证码、发票自动归到一类，跟人的邮件分开。

日历：没有月视图，只有周和年

"我觉得这很符合人类的第一反应 — 以一周的视角去看待最近发生的事情。" 年视图打开后会有一种"很震撼"的感觉。

看板：每次只能打开两列

"打开多了会让你过载。" 引导卡片做得非常舒服，每张卡片都教你下一步该做什么。

买断 + 开源 + 强制休假

"一个公司付 99 刀就可以用了。" 几乎所有产品都开源，部署到自己服务器免费。强制休假 + 利润 10-15% 分给员工。

更狠的还在后面 — 白豆翻到他们最近的一篇博客 "逃离云端"：他们算过账，自购服务器加运维一年比 AWS 省好几百万美元。AWS 利润率 60% 以上 — 这个数字本身就是答案。

李
为什么国外会有这么多这种公司，国内做不了？
白
国内也有 — 只是不知道、或者宣传少。氛围还是有点不一样 — 国内表达不多的是闷声发大财，表达多的就是那种特别浮夸的。

Chapter 13

本期启示

01

底线被抬高了

原来不好好干活就饿死，现在不好好干活也不饿死。当所有人的"默认产出"都被工具抬高，原来你以为可以靠的护城河 — 比别人深一寸 — 就不太够了。

02

名词堆砌的反面

当 AI 让每个名词的表面熟悉度都拉高，"我能把一件事完整做出来"成了越来越稀缺的证明。一个项目讲清楚 > 十个名词的简历。

03

赚钱 = 信息差 × 找对人

不是绝对的快，是相对的快。"在非英语国家教英语"比"在英语国家教英语"赚钱 — 找到具体快于谁的那群人，才是关键。

04

唱跳 Rap 都得会一点

当单点能力的中位数被工具抬高，跨界比再深一寸更划算。写作 + 演讲 + 视频 + 工程 — 不一定每一项都顶尖，但每一项都不能短板。

05

99% → 99.5% 太贵

DHH 用最直白的方式说：从 99% 到 99.5% 要付出几倍的努力。把那几倍的精力拿去做另一件事，可能就是另一个 90% — 总价值反而更高。

06

累得不一样

同样是疲惫，"自愿的累"和"为了一份钱的累"颜色不同。身体还是会表现疲惫，但是底色不一样 — 这件事自己心里清楚就好。

07

持续出现 = 击败放弃的人

很多平台的算法不是奖励"做得多好"，是奖励"还在做"。坚持出现就比一半人强 — 哪怕只是走路时顺手截一张图。

08

粉丝数 ≠ 转化能力

百万粉丝赚不到几万粉丝的钱。公众号一个粉丝抵十几个抖音粉丝。选平台不是看体量 — 是看你的内容和平台逻辑是否匹配。

09

碎片化积累胜过整段时间

素材不是大块时间堆出来的，是走路时、申请小程序时、看见有意思东西时一点一点扔进库的。最后回头写文章的时候，素材已经在那儿。

10

把"被记住"做成礼物

"每个人都会有值得学习的地方" — 把它记下来，几千天后还能调出来。最有杀伤力的不是新颖的工具，是"原来你还记得"。
