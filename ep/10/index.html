Chapter 01

从济州岛回到北京 — 火山石与海豚

"我第一次在外边见到海豚" — 摇摇的济州岛

本期摇摇从济州岛刚回来。她带回来的不只是几张游艇上看到的海豚照片，还有一些关于"地方记忆点"的小观察：

「在北京嘛，以后一直会在北京，时间还长 — 离开之后才发现，确实也没有特别多的机会去看了。老家觉得自己是山西人，山西那么多地方为什么要优先去？肯定是先看外边的。其实现在我玩的最少的地方可能就是山西了。

值得去的地方还是要尽可能多去体验。」

— 摇摇

她还讲了济州岛"三怪"的当地传说 — 喀纳斯有水怪、济州岛有"石头多、女人多"的旅游标签。两个原因都很沉：

女人多的成因

当地是捕鱼为生，男人出去捕鱼很多没回来。再加上历史上有一次政变 — 杀了大量男人，剩下女人多。

石头多的成因 — 火山喷发

"火山石以前可以带回去留纪念，后来游客越来越多就不让带了。"

火山石的另一种用途 — 鱼缸滤材

"火山石疏松多孔，可以吸附很多脏东西 — 这也是这边水可能比较干净的一个原因。"摇摇刚养了乌龟（详见 EP11），所以对这点特别敏感。

她临结束分享时给了一个让所有人都笑了的提醒 — "我最近在看那些防晒，你要是想显得不老，做好防晒，最近买了多防晒霜 — 有需求可以跟我讨论一下。"

Chapter 02

福特流水线与 AI 组织重构的争执

"老板们想搞的，是一个新的流水线 — 但要先把每一道工序的稳定性做出来"

这一周摇摇被两件事缠住 — 一件是和老板们的方向分歧、一件是和大部门里"某位负责流程重塑的同事"在知识库重构上的争执。

她对 AI 组织重构有非常具体的思考 — 这一段是她本期最浓的"工程师式"思考：

「老板们觉得 — 单点提效意义不大。组织优化和流程重塑必须并行去做。

但我觉得这两件事的顺序不能颠倒：前面这些 skills 还没有大量运用，根本不知道流程中哪一步可以删减、哪些可以合并。我们目标是大模型负责编排，但具体执行还是要靠脚本和稳定的工具。只要涉及大模型，就一定会出现不稳定因素。

所以我自己的看法是 — 让单点提效的东西先用起来，用一段时间之后，发现某个评审越来越用不上了，你就说明评审可以不用做了。」

— 摇摇

她当场跟 AI 讨论了一下福特流水线 — 想知道它对今天 AI 组织重构有没有借鉴。她得到的答案是：

「福特一开始也不是把工厂生产流程全部推翻 — 标准化、工序拆分、专业化工具 — 这些前置动作做好了之后才提出来的。

类比到现在 — 当每个人都在用 AI 做生产工作的一部分，这个东西就可以嵌到他的工作流里，每一道工序都是可控的、用统一的平台和架构，这样产物才可以层层传递不丢失。流程自然就会被重构。」

— 摇摇（与 AI 的对话总结）

她对那些"已经在裁人"的赛马公司也给了一个冷静判断 — "他们也没有真的做到理想中的 AI Native — 我觉得他们现在是在追求里面数据的好看，所以这么做。"

另一件事 — 知识库重构 — 让她遇到了"上班以来第一次被凶"：

「他想重构知识库。我倾向于不从头重构 — 在原有架构上去加 metadata、加 schema 就好了。

但他觉得现在数据质量太低、必须治理。我说 — 他们觉得知识库不好用的原因是东西太旧；东西旧的原因不就是因为新的东西没有入库吗？入库这件事推广不出去，你再建 10 个知识库效果还是一样的差。

然后他就凶我了 — 我头一回上班被凶。难受。男的他就说不是。

我觉得在公司上班太难了。」

— 摇摇

这时候摇摇老板给了她一句很金的建议：

「你不要想着去说服他 — 他的想法你就尊重他的想法，然后你就做自己想做的事情。

求同存异 — 同的东西你去他那争取资源，异的东西你找我来，我来给你撑腰。这样事情会推得更快。」

— 摇摇老板（转述）

李墨玩接了一句非常工程师式的安慰 — "工作被别人怼了没事 — 在自己的情绪上要有一个区分，不然会很累的。"

Chapter 03

寓言故事的第一次落地 — 108 个转发

上期（EP12）李墨玩立的 flag — 用寓言故事的方式去讲 GitHub 和 Git — 这周已经发出去了。结果出乎他意料：

「我也没有想到 — 那篇我就随便写了，然后好多人转发，那篇涨了我好几二三十个粉丝。我那篇有 108 个转发了 — 我也不知道为什么。这就还挺有意思的。」

— 李墨玩

顺势他把读书笔记也固定下来 — 每个礼拜天发一篇。这周日发第二篇、下周日第三篇。摇摇当场提了一个对照：

「我之前在微博上关注了一个博主，他写东西写的就特别美 — 后来他也去开班、开培训班来教这些事情。叫故园风雨前。」

— 摇摇

李墨玩说他可以走得更远："写作的还是挺赚钱的，我会开读书的社群 — 这个东西不费我很多精力。"

但小程序"翻翻"（他做的那个抽塔罗牌式的教员语录小程序）这一版又被微信审核毙了 — 原因是"娱乐化党史"。李墨玩有非常直接的判断：

「这个东西明显不是娱乐化 — 我觉得还挺好判断的。审核人员其实是故意刁难你，有可能他本身就特别讨厌这个题材。

但我们要知道 — 你说他一刀切，背后审核的那个人也是具体的人去干活。所以我后面改一版吧，改成什么学习打卡的；再不行就做成 App — 反正我现在也有网页版能用，做成 App 后面可以加很多自己的功能。」

— 李墨玩

摇摇补了一句很冷静的："审核也不一定完全是讨厌这个人，他可能也是懒 — 涉及话题敏感的，他都很敏感。"

Chapter 04

Astro × GitHub × 宝塔 — 一个 670 篇文章网站的轻量架构

这一段是李墨玩这周演示得最详细的一项 — 他给一位育儿方向的客户搭了一个 670+ 篇文章的内容网站。整套架构的关键判断是 —— "轻量化运维的方式比管理后台优先"。

架构的拓扑：

Astro 做前端 + Markdown 文件即数据

"东西特别重的话每次要改 — 还要去改数据库，特别麻烦。我就想到用 GitHub 仓库去管理文章 — 都是 Markdown 文档。"分了几个分类：AI 思考、AI 实践、产品测评、教程、读书、Thinking。

新文章流转 — AI 自动清洗 + 分类 + 打标签

"我让 AI 去读我这篇新文章 + 一个'文章更新流程'的说明文档 — 先把开头的广告、不干净的东西清洗掉，转成干净的 Markdown，再分到对应分类、打标签。"

git push 自动部署

"分类完了之后直接 git push — GitHub Webhook 同步到宝塔面板，宝塔面板的自动部署脚本就给我构建了。非常方便、比较轻。"

为什么不做后台管理系统？

"我还没有磨合出来'我平时是怎么上传文章的'这套 SOP — 在没找到一套稳定流程之前，先做轻量化运维。等流程稳定下来之后再做后台管理系统。"

这一段的设计哲学和上期"Skill 不要装太多东西、做原子化能力"是同一根线 — 先把"做这件事的流程"摸清楚，再把流程封装成产品。颠倒过来就是给自己挖坑。

他顺手秀了一下自己的排版工具的后台数据：埋点统计 UV/PV、完读率（根据滚动位置算的）、来源国家、付费用户 4-5 个、注册用户 40 个（去掉自己 39 个）、独立游客 100 多。"来源里居然有印度的、有美国的 — 我也不知道为什么。"

Chapter 05

白豆，做你的 Newsletter 吧

李墨玩看完白豆的"豆推荐"页面（已存在但他自己没更新），提了一个干脆的建议：

「我觉得白豆非常有机会去做这样一件事情 — 把你的信息源整理一下，让 AI 从里面设定一个评判标准、帮你筛选、然后分发出来。

因为创业的 follower 们做的都特别重、特别多 — 很多时候我并不需要那么多的信息，也不需要关注很多人。更重要的是有一个人帮我做了筛选的工作，然后再分发出来 — 比较省力气。

你可以把它打包成 Newsletter 或者 RSS 订阅 — 第一版就把你整理一下推给我们。」

— 李墨玩

白豆当场打开"豆推荐"页面 — 居然真的有新留言，是他高中师兄留的："底下第一个评论 — 太巧了。"

白豆顺势答应重构 — "界面又比较丑、bug 特别多、更新起来很麻烦"。李墨玩给了一个具体建议："你到时候就放联系方式、不要加评论功能 — 容易被攻击，给你发一堆垃圾"。

这就是本期一个非常隐性的主题 — AI 时代，"做筛选 + 分发"比"做内容"更稀缺。一个有 taste 的人 × 一个被时间炼过的推荐列表 = 一个可以被别人订阅的资产。

Chapter 06

切换出发地的状态机 — 抄了一下携程

白豆上次给李墨玩提的一个看起来"超简单"的 bug：高铁 ↔ 飞机切换的时候，出发地和到达地要不要同步？

「我本来一开始想 — 这玩意还挺简单的，不就加个历史记录吗？我以为半小时搞定。结果一仔细搞的时候发现挺复杂的。」

— 李墨玩

他列了几个真实场景里的状态分叉 — 全是非工程师不会想到的边界：

Case 1 — 一边有车、另一边没车

"从我们家江西鹰潭到北京 — 有高铁；切到飞机，这里没飞机场。怎么办？"

Case 2 — 子站没机场，要映射到父城市

"从宝坻到上海有高铁直达 — 宝坻属于天津市但没机场。切到飞机，必须把宝坻映射回'天津'这座城市。"

Case 3 — 来回切的状态流转

"当我切到飞机后又改了北京到上海，再切回高铁 — 状态怎么同步？分叉太多了。"

Case 4 — 所有逻辑都失败时的兜底

"必须有个兜底的逻辑 — 不然用户就卡死了。"

他最后是直接逆向了携程：

「携程它不是可以切吗？我订机票选了北京到上海，切到火车票，它怎么流转的 — 你就试几个 case，就知道它是怎么做的。就有点像人家说逆向别人的算法、逆向别人的代码不就是这样吗？」

— 李墨玩

原本半小时的事，最后搞了一个多小时。这个故事的精度在于 — "看起来简单的 bug"其实是状态机的设计题；最快的解法是看一个被验证过的产品是怎么做的，而不是自己从零推。

Chapter 07

字节众测的 Skill — 200 元提效的标准化流程

这是李墨玩这周演示得最"工程化"的部分 — 他把字节模型众测平台的整套流程封装成了一个 Skill，单次提效大约 200 元（约 1 个任务的报酬）。

他事先有一条踩坑教训 — "千万不要做从 0 到 1 的"：他之前给一个克隆"小米声音克隆"页面的任务做了从 0 搭建的网站，结果测试时累死了 — 要测灯、测配置、测 API 联通、测音色克隆等各种东西。

所以他这次的 Skill 思路是 — 挑"非常简单又能跑满测评轮次"的任务类型，比如评论审核：

任务设计 — 让 AI 帮你写任务书

"任务书也是 AI 帮我写的。我会给他一个参考 — 各种任务类型：注册登录、小程序转 Web、接入第三方 API、文件上传下来导出、搜索等等。"

挑能跑满轮次的任务

"一个纯前端的功能跑不满测评要求的轮次，就会失效。我选的'前台评论 + 后台审核'这种 — 简单、又能跑满。"

四个模型同时启动端口

"它一个任务要跑四个模型 — 我让 AI 同时把四个模型端口全部启动。本地点点点就去访问，我手动测试的同时，让 AI 在后面跑自动化测试。"

截图文件名 = 评价

"截图也有个技巧 — 把截图的文件名作为评判的评论。AI 后面看到文件名就知道我们的评价是什么样的，他就会去写一个测评报告。"

评分表 + 一句话评论 + 轨迹评论

"对应每个项目的综合评分、一句话评论、上传截图、轨迹分析（这个模型的代码哪个模块有问题） — 本地复制到平台只需 Ctrl+F 搜索定位，效率非常高。"

这一整套就是本期主题 — "把流程当作产品" 的最纯粹示范。一次众测的报酬不多，但当流程被封装成 Skill 之后，这个流程本身就成了可复用的资产。

Chapter 08

Anki 就是你的知识库 + 微信附件的小发现

"我的知识库就是 Anki — 直接用 Anki Connect 搜东西"

这一段从摇摇问的一个问题开始 — "你的 Anki 可不可以导到 Agent 里面，作为它检索的输入？"

「非常有道理 — 可以的。我目前搜东西就直接把 Anki 打开 — 有一个 Anki Connect，它就直接能去里面搜。相当于有人问我的知识库是什么，我就跟他分享的是 Anki。

一是它能锻炼我自己的脑子，二是它是个知识库 — 把资料存里面。不过你分享对我的启发 — 我可以建立备份同步的东西，直接导到 ima 知识库 — 后续开一些付费课程的时候，可以当作福利的一部分。」

— 白豆

白豆这周还分享了两个看起来很小、其实很关键的"折叠屏的优势"小发现：

填表单可以双面显示

"折叠屏的优势 — 是不断探索的过程。填表单的时候它只有一面，我在电脑上填很方便；但有时候我又想去 ⋯"（折叠开就同时能看）

微信附件多选的"非 bug"

"我之前一直以为 — 上传多张图片时只能点一次附件传一张，是必须的流程。后来用浏览器打开链接，会发现可以选中多张图片一次性上传 — 这是微信接口的问题，不是必须。当记录多张图片附件的时候，特别麻烦 — 换一个打开方式，没想到真的不一样。"

"出租屋要不要自己装"的小决定

"住的时间长，可以稍微装一下 — 空间还挺自由的，想买什么就装了，退租的时候恢复就行。"

这些都是看起来"很小"的发现，但它们都是同一个心法 — 默认的工具行为不一定是最优的，换一个口子就有新答案。

Chapter 09

4 块钱做完的多语言案情视频

"6 个视频，一共花了 4 块钱 API — 用的 MiniMax 的"

白豆给某律所客户做的新需求 — 客户要跟国外律师开线上会，每次都要同步案情（背景、判决、商标争议等），同一个东西要用不同语言讲好多遍 — 特别麻烦。

他用 Codex 接 MiniMax TTS + HTML/CSS 动画做了一个完整的多语言案情介绍视频生成工作流：

输入：客户给的 PPT

"他给我一个 PPT — 我跑了一下，还真我觉得效果还不错，不过可以优化。多语言。"

没用 Remotion，用纯 HTML/CSS 动画

"我啥 Skill 都没给他 — 就是让他描述了我想要的，让他用 HTML 然后去做这种动画，本质上就是 ffmpeg 那种工具做的。"

音频用 MiniMax 自己写调用 + 大约 3 分钟介绍

"声音是让他自己写调用模型，自己去写一个大概 3 分钟左右的介绍，用 MiniMax 的 API 去调的音频。"

成本：6 个视频共 4 元 API 费

"整个下来做了大概 6 个视频，一共花了 4 块钱 API。"

李墨玩当场建议他用小米通义的 TTS — "它不要钱"。白豆记下了下次试试。

白豆下一步还想加上 B 站那种数字人 + 克隆客户自己的声音，让案情视频"有亲近感"。他研究了美团开源的 LongCat Video Avatar — 本地配置跑不动，最后是去 AutoDL 租 GPU — 选了一个已经把环境配置好的镜像，几块钱一小时，充了 20 块钱试。

「之所以选这个平台，是因为它推荐的平台上已经有人把这些知名项目的环境都给配置好了 — 你现在只要选中项目，直接把它镜像拉过来就行。几块钱一个小时 — 创建我这几个视频可能就十几分钟，成本非常低。一旦走通了，能给客户创造很大的效益。」

— 白豆

Chapter 10

饭局上的老板 — 手环、二类器械与"借东风"

白豆这周一直在饭局 — 老板带他出去见医疗投资圈的人。他做了一段非常细的"老板观察"：

「我发现老板的一个特点 — 有时候他有了一个想法之后，就会一遍一遍地跟身边的人、他的朋友去叙述这个东西，带来了很多激情。

有时候我会觉得他可能是夸大 — 不是夸大吧，他在描述他的愿景。真的会有一些朋友资源比较多，真的就有别的方面去合作。」

— 白豆

具体的合作雏形 — 从白豆做的那个简单的 NFC 手环出发，老板想做一个能测健康数据的手环（二类器械），跟公司 IP 形象联动起来：

二类器械 + 高校院士科研合作

"国内目前有两家可以做 — 不用显示屏、可以测比较准确的数据，跟某高校的院士进行科研合作，二类器械拿了证。"

用文化叙事包装医疗数据

"基本上都是很标准的数据 — 联系起来之后用一些'仙侠类的词'解读，再给你推荐一些食品。不是医疗行为，是诊断 + 推荐。"

银行扶持基金

"还有民生银行的高管来 — 他们内部有一些扶持计划，可能会出 100 万的基金直接送给你，不是要占股权 — 只不过后期你要用一些金融服务可能要用他们家。"

白豆的感受是 — "老板天马行空，但真的能碰撞出来一些别的东西"。今天见的几个人都在医疗圈、投资圈相当有名，能看到的不只是一些可能性，更是这一类人之间"借东风"的速度。

Chapter 11

律师 MCP × 警察朋友的刷题工具 — 高价值垂类

李墨玩在线上看到一个律师查裁判文书网的 MCP 产品 — 他抓住机会给白豆做了一个非常具体的方向建议：

「我跟你说法律有非常多的场景。很多律师打官司要查裁判文书网的案例做参考 — 人工查很慢，他就用 MCP 去查这些东西。有点像知识库。

你客户至少是中高层 — 他懂这些专业的东西、有一些垂类的场景。不一定要"为他做" — 是说跟他一起合作，解决他的需求，跟他一起入股、分成；让他利用律师圈里的资源帮你出售、推广 — 他又是真实用户。

律师都比较有钱 — 这是一个非常好的场景。」

— 李墨玩

白豆顺势分享了今天刚做的另一个东西 — 给警察朋友做的考试刷题工具：

「之前给大家分享了我那个刷题的东西，没想到这个东西就又来了 — 他要考试。我准备到时候做，就真的可以这样了。

我接了一个 DeepSeek — 把他给的 Word 文档全都做成了可交互式的可刷题的东西，嵌入了一个 DeepSeek 去讲解。速度有点慢，但他就不用去复制。整个成本还比较低。

20 分钟不到 10 分钟，跟 AI 说了几句，就做好了。」

— 白豆

摇摇当场识别出了它的形态：

「你的组织形式 — 和你的某一个 Anki 的模板特别像。」

— 摇摇

李墨玩立刻"借鉴"了这条思路 — "我一直想做帮我复习公考的东西，我后面就用你这个思路 — 我去搭一个针对于公考的场景，然后我就发到小红书，会有很多人来买"。

本期最浓的一段商业暗线在这 — "高价值的垂直 + 真实用户 + 你做过的模板的复用" = 一个可以反复出售的产品。律师 MCP、警察刷题、公考刷题 — 都是这条公式的实例。

Chapter 12

小米通义的"积分通货膨胀" + 做客嘉宾入住指南

"做客嘉宾的入住指南，让我们这个节目变得正规化"

白豆讲他买了一个月小米通义的会员 — "40 多个亿的 Token，特别便宜"。这一句话让摇摇和李墨玩同时笑了 — 摇摇说："那天李墨玩跟我说的时候我也愣了一下，他好坏。"

「不是 — 他那是积分。有个换算比例的。比如 80 块钱送的套餐大概是 17 个亿 Token —— 100 多万积分换算 17 亿 Token 左右。

通货膨胀了 — Token 也他妈通货膨胀。罗福利真的骚，他搞的这些操作。」

— 李墨玩

顺势他们也聊了 Codex 5x 套餐这次重置加倍的事情：白豆是 5x，"这次他好像是按 10 倍的算，相当于额度多了"。Codex 现在比 Cloud 性价比高很多 — 白豆这一个月已经没有续 Cloud 了。

更深一些的看法 — "5.5 Pro 是数学家专属"：

「OpenAI 有个工程师讲 — 什么样的人在用 5.5 Pro？数学家。数学家喜欢一个问题想好久无所谓 — 他有的是时间，所以他就用。」

— 李墨玩（转述 OpenAI 工程师）

临散场，李墨玩提了一件特别"产品化"的事 — 给做客嘉宾做一个入住指南 SOP：

「让他来了之后，给他一个本周小总结的模板 — 让他自己总结也行，不用特别正式。

这样有几个好处：第一，减少沟通成本，他不用问东问西、你也不用解释那么多；第二，他来了知道干什么，会有点参与感。

我们就可以逐渐迭代这个事情，让它变得有 SOP 流程 — 后面就可以形成这样的东西。我有时间，下周我来搞。」

— 李墨玩

摇摇当场提了具体协作方式 — "你可以发一个 Git 出来，然后一起搞"。李墨玩答："仓库的 pull request 你同意就好了 — 行，那就这样。"

这一段是本期主题 — "把流程当作产品" — 最自指的一次应用：连我们自己的节目本身，都可以被产品化为一套有 SOP 的流程。

Chapter 13

本期启示

01

福特流水线不是先推翻，是先规范

标准化、工序拆分、专业化工具 — 这些前置动作做好了之后，流程重塑才能真的落地。AI Native 组织一样 — 先让单点提效跑起来，再谈推翻。

02

求同存异，不要试图说服

"他的想法你就尊重他的想法 — 同的去他那争资源，异的找老板撑腰。" 上班最累的不是干活，是想说服每个人都同意你。

03

寓言故事是被高估的写作技能

108 个转发可能比你想的简单 — 用故事讲技术概念是个被低估的"通用型技能"，特别是讲一个被反复解释的老概念时。

04

轻量运维优先于管理后台

在你的 SOP 还没磨合出来之前，"GitHub + Webhook + 静态站"比"Mysql + 后台" 优先。后台是产品，流程是基础 — 顺序不能颠倒。

05

"做筛选 + 分发"比"做内容"更稀缺

AI 时代信息流爆炸了 — 一个有 taste 的人 × 一份被时间炼过的推荐 = 一个可以被订阅的资产。Newsletter 不是过去式，是新机会。

06

看起来简单的 bug 是状态机题

切换出发地这种"半小时搞定"的 bug，往往背后是 4 条分叉 + 1 个兜底。最快的解法 — 不是自己推，是逆向一个被验证过的产品。

07

把流程当作产品

众测 200 元 / 任务 — 一次性投入封装成 Skill 之后，流程本身就是可复用资产。下次只要套上去就出结果 — 这是本期最重要的一条。

08

知识库不是数据库，是"你能找到的过去的你"

"我的知识库就是 Anki。" Anki Connect 直接搜 — 比任何向量库都贴近你的实际使用方式。福利就是付费课程的"已经在你脑子里的那部分"。

09

4 块钱的视频已经能用

HTML/CSS 动画 + MiniMax TTS + Codex 编排 — 6 个多语言案情视频 = 4 块钱 API。"成本非常低 + 给客户创造很大效益" — 这是新工作的样子。

10

租 GPU 比配本地环境快

AutoDL 上别人已经把镜像配好了 — 几块钱一小时直接拉来用。"自己折腾本地"是一个被高估的工程师骄傲。

11

老板的"夸大"不一定是夸大

他在描述愿景。"真的会有朋友资源比较多，真的有别的方面去合作。" 反复地讲愿景 — 本身就是一种筛选与吸引。

12

高价值垂类 + 真实用户 + 你的模板

律师 MCP / 警察刷题 / 公考刷题 — 同一套配方，不同的细分。"你客户至少是中高层" + "他懂专业" + "他能帮你卖" — 比靠流量便宜得多。

· · ·
Epilogue

把流程当作产品 — 也包括我们自己

本期最有意思的一个反复 — 大家都在用同一个动作（虽然没明说）：把"做完一件事的过程"，变成下次可以套上去的流程。

  • 摇摇说 — "福特流水线不是先推翻，是先标准化"。
  • 李墨玩的 670 篇文章网站 — 用 git push 把"上传文章"标准化。
  • 李墨玩的 字节众测 Skill — 把"做一次众测"标准化成可复用的 200 元提效。
  • 白豆的 多语言案情视频流水线 — 把"3 分钟的案情介绍"标准化成 4 块钱的产品。
  • 白豆给警察朋友的 刷题工具 — 套用了之前 Anki 模板的"刷题 + 讲解"流程。
  • 李墨玩提议的 做客嘉宾入住指南 SOP — 把"我们这个节目"本身也变成了一个有流程的产品。

这条线和上期"让记忆成为一种选择"一脉相承 — 当 AI 把"做一件事"的成本砍到几乎免费，剩下的稀缺是"能把这件事变成下次可以反复做的"。

临散场，摇摇给白豆下了一个提醒 — 这一句话也是本期最像"姐姐"的一句：

「白豆你一定得找时间休息一波 — 你状态太差了。」

— 摇摇

白豆答："谢谢摇摇提醒，我确实好好休息。"
